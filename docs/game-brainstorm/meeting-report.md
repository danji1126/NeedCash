## 1. 현황 분석

### 기존 게임 목록

| 게임 | 유형 | 핵심 기술 | 코드량 |
|------|------|-----------|--------|
| Dice Roller | 운/확률 | Math.random, framer-motion 3D 회전 | 133줄 |
| Lotto Pick | 운/확률 | Fisher-Yates 셔플, 순차 애니메이션 | 163줄 |
| 동물상 찾기 | AI 분석 | TensorFlow ML, WebRTC, Canvas | 723줄 |

### 포트폴리오 강점
- 기술 레벨 그라데이션 존재 (기초 state ~ ML 모델 통합)
- 동물상 찾기의 높은 참여도 (셀프 콘텐츠 생성 패턴)
- 일관된 디자인 언어와 UX 패턴

### 포트폴리오 약점 (3명 전문가 공통 지적)
- **운 기반 편중**: 3개 중 2개가 순수 랜덤 (사용자 의사결정 없음)
- **스킬 기반 게임 전무**: 사고력/전략/반사신경을 요구하는 게임 부재
- **진행 감각 부재**: 반복해도 축적되는 것이 없음
- **SNS 공유 메커니즘 부재**: 결과 공유/비교 기능 없음
- **Canvas/WebGL 기술 데모 부족**: framer-motion 외 시각적 기술 시연 없음

## 3. 교차 투표 및 최종 선정

### 점수 산정 (1위=5점, 2위=4점, 3위=3점, 4위=2점, 5위=1점)

| 게임 | 프론트엔드 | UX/프로덕트 | 트렌드/시장 | **총점** |
|------|:----------:|:-----------:|:-----------:|:--------:|
| **반응속도 테스트** | 5 (1위) | 4 (2위) | 5 (1위) | **14** |
| **색깔 구별 테스트** | 4 (2위) | 5 (1위) | 2 (4위) | **11** |
| **타이핑 속도** | 1 (5위) | 3 (3위) | 3 (3위) | **7** |
| **색상 기억력** | - | 2 (4위) | 4 (2위) | **6** |
| **숫자 퍼즐** | 2 (4위) | 1 (5위) | 1 (5위) | **4** |
| **카드 뒤집기** | 3 (3위) | - | - | **3** |

### 최종 선정: 반응속도 테스트 (Reaction Time Test)

**선정 근거:**
1. **전원 1~2위 추천** - 3명 전문가 모두 최상위 추천 (총 14점, 압도적 1위)
2. **최소 구현 비용** - ~200줄, 난이도 1.5/5, 외부 의존성 0
3. **최대 바이럴 잠재력** - Human Benchmark 틱톡 바이럴 검증된 포맷
4. **기존 게임과 완전한 차별화** - 유일한 "반사신경/스킬" 기반 게임
5. **모바일 최적** - 전체화면 터치, 한 손 조작
6. **빠른 피드백 루프** - 30초 플레이 → 즉시 결과 → 재도전

## 4. 선정 게임 상세 기획서

### 기본 정보

| 항목 | 내용 |
|------|------|
| **이름** | Reaction Test (반응속도 테스트) |
| **슬러그** | `reaction` |
| **한줄 설명** | "당신의 반응속도는 몇 ms? 지금 테스트하세요" |
| **아이콘** | bolt / zap (번개) |
| **타겟 플레이 시간** | 30초 ~ 1분 |

### 게임 흐름

```
[대기 화면] 빨간 배경 + "초록색이 되면 클릭하세요!" 안내
     |
     v (랜덤 2~5초 딜레이)
[GO!] 초록 배경 전환 → 사용자 클릭
     |
     ├─ 정상 클릭 → XXXms 표시 → 다음 라운드
     └─ 너무 빨리 클릭 → "Too Early!" 경고 → 재시도
     |
     v (5회 반복)
[결과 화면] 평균 ms, 최고/최저, 등급, 비유 칭호
     |
     ├─ "다시 도전" → 새 세션
     └─ 히스토리에 기록 저장
```

### 등급 시스템

| 등급 | 범위 | 칭호 |
|:----:|------|------|
| S | < 200ms | "번개 반사신경" |
| A | 200~250ms | "매의 눈" |
| B | 250~300ms | "민첩한 고양이" |
| C | 300~400ms | "평범한 인간" |
| D | 400~500ms | "느긋한 거북이" |
| F | > 500ms | "졸린 나무늘보" |

### 기술 구현 방향

| 항목 | 방식 |
|------|------|
| **상태 관리** | `useState` - phase, 시작 타임스탬프, 결과 배열 |
| **타이밍 측정** | `performance.now()` (밀리초 정밀도) |
| **랜덤 딜레이** | `setTimeout` + `useRef`로 cleanup |
| **애니메이션** | framer-motion `backgroundColor` 전환 |
| **결과 통계** | 평균, 중앙값, 최고, 최저 계산 |
| **히스토리** | 기존 패턴 동일 (최근 10개) |
| **모바일** | 전체 화면 터치 영역, 한 손 조작 |

## 5. 향후 확장 로드맵

새 게임을 추가한다면 아래 순서를 추천합니다:

| 우선순위 | 게임 | 카테고리 | 기존 게임과의 시너지 |
|:--------:|------|----------|---------------------|
| 1 (선정) | 반응속도 테스트 | 반사신경/스킬 | 운 기반과 대비되는 스킬 게임 |
| 2 | 색깔 구별 테스트 | 감각/시각 | "감각 테스트" 시리즈 확장 |
| 3 | 타이핑 레이서 | 입력/스킬 | 개발자 타겟 게임 |

이 3개를 추가하면 게임 허브가 **4가지 카테고리**로 균형 잡힌 포트폴리오가 됩니다:
- **운 기반**: Dice Roller, Lotto Pick
- **AI 분석**: 동물상 찾기
- **반사신경/감각**: 반응속도 테스트, 색깔 구별 테스트
- **스킬**: 타이핑 레이서

## 부록: 전문가 분석 원문 요약

### A. 프론트엔드/게임 전문가
- 5개 아이디어 모두 외부 라이브러리 0, 번들 사이즈 0KB 추가
- 기존 패턴(useState/framer-motion/히스토리/단일 파일)과 100% 호환
- Reaction Test를 1순위 추천 (가장 간단한 구현 + 모바일 최적 + 완전한 차별화)

### B. UX/프로덕트 전문가
- 현재 게임의 핵심 약점: 스킬 기반 0개, 진행 감각 부재, SNS 공유 부재
- Color Sense Test를 1순위 추천 (바이럴 잠재력 + 원온보딩 + 중독성)
- Reaction Test를 2순위 추천 (공유성 + 간단 구현 + 빠른 피드백)
- 2개 추가 시 "감각 테스트" 카테고리로 묶으면 정체성 강화

### C. 트렌드/시장 전문가
- 2024-2025 웹 미니게임 트렌드: Wordle류, 타이핑, AI 게임, 일일 챌린지
- Human Benchmark 틱톡 바이럴 (수천만 조회) - 반응속도 테스트 검증된 포맷
- "Is my blue your blue?" 컬러 테스트 SNS 대유행 - 색감 테스트 시의성
- Reaction Test를 1순위 추천 (빠른 구현 + 높은 바이럴 + 검증된 포맷)
