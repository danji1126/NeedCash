# Plan: game-content (PDCA-3)

> **Feature**: game-content
> **상위 계획**: `docs/brainstorm/adsense-approval.plan.md` > PDCA-3

> **우선순위**: P0 (즉시)
> **의존성**: PDCA-2 (seo-infrastructure) 완료

---

## 1. 목적

6개 게임 페이지의 "씬 콘텐츠(thin content)" 판정을 해소한다. 현재 게임 상세 페이지는 제목 + 한 줄 설명 + 게임 컴포넌트만 존재하여 구글봇이 가치 있는 콘텐츠로 인식하지 못한다. 각 게임 페이지에 서버 렌더링되는 정적 텍스트 콘텐츠를 추가하여 독립적인 검색 가치를 부여한다.

---

## 2. 현재 상태 분석

### 게임 상세 페이지 구조 (현재)

```
[Back 링크]
[Game 라벨]
[아이콘]
[H1: 게임 제목]
[한 줄 설명]
[구분선]
[게임 컴포넌트] ← 클라이언트 렌더링, 봇이 읽을 수 없음
```

### 게임별 텍스트 콘텐츠 현황

| 게임 | slug | 페이지 정적 텍스트 | 컴포넌트 내 텍스트 | 총 콘텐츠 |
|------|------|-------------------|-------------------|----------|
| Dice Roller | dice | 제목+설명 (1줄) | UI 라벨만 | 매우 부족 |
| Lotto Pick | lotto | 제목+설명 (1줄) | UI 라벨만 | 매우 부족 |
| 동물상 찾기 | animal-face | 제목+설명 (1줄) | 동물 설명 200줄+ | 부분적 |
| Reaction Test | reaction | 제목+설명 (1줄) | 규칙+등급 15줄 | 부족 |
| Color Sense | color-sense | 제목+설명 (1줄) | 규칙+등급 15줄 | 부족 |
| Color Memory | color-memory | 제목+설명 (1줄) | 규칙+등급 20줄 | 부족 |

### 핵심 문제
- 게임 상세 페이지의 **서버 렌더링 텍스트**가 제목+설명뿐
- 게임 컴포넌트 내부 텍스트는 `"use client"`이므로 **구글봇이 읽기 어려움**
- SEO에서 가치 있는 콘텐츠로 인식되려면 **서버 렌더링 정적 텍스트 1000자+** 필요

---

## 3. 요구사항

### FR-01: 게임 콘텐츠 데이터 구조

- `lib/game-content.ts` 파일 생성
- 6개 게임 각각의 콘텐츠 데이터 정의:
  - `introduction`: 게임 소개 (200자+)
  - `howToPlay`: 플레이 방법 (step 배열)
  - `scoreGuide`: 점수/결과 해석 (항목 배열)
  - `background`: 과학적/교육적 배경 (300자+)
  - `faq`: FAQ 질문-답변 쌍 (3개+)
- **완료 기준**: 6개 게임 모두 데이터 정의, 게임당 총 1000자+

### FR-02: 게임 콘텐츠 컴포넌트

- `components/game/game-content-section.tsx` 서버 컴포넌트 생성
- 게임 컴포넌트 **아래**에 콘텐츠 섹션 렌더링:
  - H2: 게임 소개
  - H2: 플레이 방법 (순서 리스트)
  - H2: 결과 해석 가이드 (표 또는 리스트)
  - H2: 알아두면 재미있는 이야기 (과학적 배경)
  - H2: 자주 묻는 질문 (아코디언/토글)
- **완료 기준**: 서버 렌더링됨, HTML 소스에 텍스트 존재

### FR-03: 게임 상세 페이지 통합

- `app/game/[slug]/page.tsx` 수정
- 게임 컴포넌트 아래에 `GameContentSection` 추가
- slug 기반으로 해당 게임의 콘텐츠 데이터 로드
- **완료 기준**: 6개 게임 페이지 모두 콘텐츠 섹션 표시

### FR-04: 관련 게임 추천

- 각 게임 콘텐츠 하단에 "관련 게임" 섹션
- 현재 게임 제외 2-3개 게임 카드 표시
- **완료 기준**: 각 페이지에 2개 이상 내부 링크

## 4. 게임별 콘텐츠 방향

| 게임 | 소개 키워드 | 과학적 배경 | FAQ 주제 |
|------|-----------|-----------|---------|
| Dice Roller | 확률, 주사위 시뮬레이션 | 확률론 기초, 큰 수의 법칙 | 공정한 주사위란?, 확률 분포 |
| Lotto Pick | 랜덤 번호, 로또 시뮬레이션 | 난수 생성 원리, 조합 수학 | 당첨 확률은?, 번호 선택 전략 |
| 동물상 찾기 | AI 얼굴 분석, 동물상 테스트 | 얼굴 인식 AI 원리 | 정확도는?, 사진 보관 여부 |
| Reaction Test | 반응속도 측정, 인지능력 | 신경전달 속도, 반응시간 과학 | 평균 반응속도는?, 향상 방법 |
| Color Sense | 색감 테스트, 색각 능력 | 추상체와 색각, 테트라크로맷 | 색맹과 다른 건가?, 훈련 가능? |
| Color Memory | 패턴 기억, 작업기억 | 작업기억 용량, 밀러의 법칙 | 기억력 향상법?, 사이먼 게임 |
