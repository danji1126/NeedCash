# 콘텐츠 확장 브레인스토밍

> NeedCash 프로토타입 허브에 새로운 콘텐츠를 추가하여 바이럴 성장, 재방문율, 콘텐츠 간 시너지를 강화하기 위한 전문가 패널 브레인스토밍 결과

## 현재 상황

| 섹션 | 콘텐츠 | 상태 |
|------|---------|------|
| Blog | D1 기반 CMS, 게시글 CRUD | 운영 중 |
| Game | 6종 (주사위, 로또, 동물상, 반응속도, 색감, 색기억) | 운영 중 |
| Resume | 5개국어 (ko, en, ja, th, vi) | 운영 중 |
| About/Privacy/Terms | 정적 페이지 | 완료 |

## 전문가 패널 분석

### 크리스텐슨 (JTBD 분석)

현재 충족되는 Jobs-to-be-Done:
- **심심풀이** → 게임 6종 (가볍게 시간 보내기)
- **자기 테스트** → 반응속도, 색감, 기억력 (내 능력 측정)
- **채용/네트워킹** → 이력서 (개발자 포트폴리오)
- **학습/정보** → 블로그 (기술 콘텐츠)

빠져있는 JTBD:
1. **"친구와 비교하고 싶다"** → 결과 공유/랭킹 시스템 부재
2. **"매일 올 이유"** → 일일 챌린지/스트릭 없음
3. **"실용적 도구"** → 순수 유틸리티 도구 없음
4. **"개발자로서 성장"** → 인터랙티브 학습 콘텐츠 없음

### 고딘 (바이럴 & 공유 가능성)

현재 게임들은 재밌지만 "이거 해봐!"라고 보내고 싶은 동기가 부족함.

공유 유발 콘텐츠 요건:
- 결과를 한눈에 보여주는 카드형 이미지/텍스트
- "내가 얼마나 잘했는지" 자랑할 수 있는 수치/등급
- 클릭 한 번으로 복사되는 공유 링크+결과

바이럴 잠재력 높은 새 게임:
| 게임 | 설명 | 바이럴 포인트 |
|------|------|--------------|
| 타이핑 스피드 | WPM 측정 + 한/영 | "내 타이핑 속도 XXwpm!" 공유 |
| 수학 암산 | 시간 내 계산 퀴즈 | "30초에 15문제!" |
| 퍼스널리티 퀴즈 | 성격 유형 테스트 | 가장 높은 공유율 |

### 드러커 (사용자 가치 & 정체성)

"프로토타입 허브"라는 정체성이 좋지만, 방문자에게 "이 사이트만의 가치"가 명확해야 함.

개발자 유틸리티 도구의 역할:
- 게임으로 유입 → 도구로 재방문 → 블로그로 체류
- 유틸리티는 "재방문 앵커" 역할
- 매일 사용하는 실용 도구 = 북마크 동기

추천 유틸리티:
| 도구 | 설명 | 가치 |
|------|------|------|
| JSON Formatter | JSON 정렬/검증/변환 | 매일 쓰는 실용 도구 |
| Base64 Encoder | 인코딩/디코딩 | 간단하지만 자주 필요 |
| Color Palette | 색상 조합 생성기 | 디자인 작업 시 재방문 |

인터랙티브 학습 콘텐츠:
| 콘텐츠 | 설명 |
|--------|------|
| 알고리즘 시각화 | 정렬/탐색 알고리즘 애니메이션 |

### 메도우즈 (시스템 시너지)

```
현재 시스템 (약한 연결):
  Blog ──(약)── Game (relatedBlog만 일부 존재)
  Resume ──(없음)── Blog
  Resume ──(없음)── Game

제안 시스템 (강한 피드백 루프):
  Blog ←──→ Game (게임 관련 블로그 + 블로그에서 게임 임베드)
    |            |
  Tools ←──→ Resume (도구 = 기술 시연, 이력서에 링크)
    |
  Daily Challenge (매일 방문 유도)
```

시너지 강화 방안:
- **Blog ↔ Game**: 게임 원리 설명 블로그 + 블로그 내 게임 링크
- **Resume ↔ Tools/Game**: "이 사이트의 모든 프로토타입을 직접 만들었습니다" — 포트폴리오 증거
- **Tools ↔ Blog**: 도구 사용법 블로그 → 도구 페이지 유입
- **Daily Challenge**: 매일 랜덤 게임 선정 + 방문 스트릭 = 재방문 루프

## 구현 계획 요약

### Phase 1: 게임 결과 공유 (바이럴 루프 핵심)

공유 유틸리티 + ShareResult 컴포넌트를 만들고 기존 6개 게임에 통합.

**새로 만들 파일**:
- `lib/share.ts` — 공유 텍스트 생성 + 클립보드 복사 유틸리티
- `components/game/share-result.tsx` — 공유 버튼 UI 컴포넌트

**수정할 파일**:
- 기존 게임 6개 컴포넌트 (결과 화면에 ShareResult 추가)

공유 텍스트 예시:
```
[NeedCash] Reaction Test
등급: S · 번개 반사신경
평균: 187ms
https://needcash.dev/game/reaction
```

---

### Phase 2: 타이핑 스피드 게임

기존 게임 등록 패턴(5단계) 그대로 따름.

**게임 설계**:
- 한/영 선택 → 60초 타이머 → WPM + 정확도 측정
- 문자별 정오답 색상 표시 (초록=정답, 빨강=오답, 회색=미입력)
- 텍스트 코퍼스: 한국어/영어 짧은 문단 3-5개씩 파일 내 상수

**등급 기준**:
| 등급 | WPM | 설명 |
|------|-----|------|
| S | 100+ | 프로 타이피스트 |
| A | 80-99 | 빠른 타이핑 |
| B | 60-79 | 평균 이상 |
| C | 40-59 | 평균 |
| D | 20-39 | 연습 필요 |
| F | <20 | 초보 |

**파일 변경**:
- 새 파일: `components/game/typing-game.tsx`
- 수정: icons.tsx(`keyboard`), constants.ts, game-content.ts, [slug]/page.tsx

---

### Phase 3: 개발자 유틸리티 도구 섹션 (/tools)

새로운 최상위 라우트 `/tools` 추가. NAV_LINKS 변경만으로 4개 디자인 헤더 모두 자동 반영.

**도구 목록**:
| 도구 | 설명 | 구현 방식 |
|------|------|----------|
| JSON Formatter | JSON 포맷/검증 | JSON.parse + stringify |
| Base64 Encoder | 인코딩/디코딩 | btoa/atob (UTF-8 안전) |
| Color Palette | 색상 팔레트 생성 | HSL 수학 기반 (외부 라이브러리 없음) |

**파일 변경**:
- 새 파일: `app/tools/page.tsx`, `app/tools/[slug]/page.tsx`, 도구 컴포넌트 3개
- 수정: constants.ts(NAV_LINKS, FOOTER_SECTIONS, TOOLS), icons.tsx, sitemap.ts

---

### Phase 4: 수학 암산 게임

Phase 2와 동일한 게임 등록 패턴.

**게임 설계**:
- 난이도 선택: easy(1-20, +−) / medium(1-50, +-×÷) / hard(1-100)
- 60초 제한, 연속 정답 스트릭 보너스
- 나눗셈: b와 answer를 먼저 생성 → a = b × answer (정수 결과 보장)

**등급 기준**: S(30+), A(20-29), B(15-19), C(10-14), D(5-9), F(<5)

**파일 변경**:
- 새 파일: `components/game/math-game.tsx`
- 수정: icons.tsx(`calculator`), constants.ts, game-content.ts, [slug]/page.tsx

---

### Phase 5: 콘텐츠 간 연결 강화

- **Blog → Game**: 블로그 본문에 관련 게임 마크다운 링크 포함 (DB 스키마 변경 불필요)
- **Game → Blog**: 이미 `relatedBlog` 필드 존재 — 실제 블로그 포스트 작성
- **Tools → Blog**: Tool 인터페이스에 `relatedBlog?` 필드 추가
- **Resume → Portfolio**: 이력서 페이지에 GAMES/TOOLS 링크 그리드 추가

### Phase 6: 데일리 챌린지

**구현**:
- 날짜 기반 시드 난수 생성기(Mulberry32 PRNG)로 매일 같은 게임 선정
- localStorage로 방문 스트릭 추적
- 홈페이지 위젯: "오늘의 챌린지" 카드 + 스트릭 카운터

**파일 변경**:
- 새 파일: `lib/daily.ts`, `components/home/daily-challenge.tsx`
- 수정: `components/home/home-page.tsx`

### Phase 7: 알고리즘 시각화

`/tools/sort-visualizer`로 배치. Phase 3 완료 후 진행.

**구현**:
- 버블/퀵/병합 정렬 3가지 알고리즘
- 전체 단계 사전 계산 → setInterval로 애니메이션 재생
- 컨트롤: 알고리즘 선택, 배열 크기(10-100), 속도, Play/Pause/Step
- 시각: 세로 막대 + 상태별 색상 (비교중=노랑, 정렬완료=초록, 피봇=빨강)

**파일 변경**:
- 새 파일: `components/tools/sort-visualizer.tsx`
- 수정: constants.ts(TOOLS), icons.tsx(`chart-bar`), [slug]/page.tsx

### Phase 8: 퍼스널리티 퀴즈

`/game/quiz`로 배치.

**구현**:

- 4가지 유형: 탐험가/창작자/분석가/연결자
- 결과: 유형 이모지 + 설명 + 특성 목록 + ShareResult 통합

**파일 변경**:
- 새 파일: `components/game/personality-quiz.tsx`
- 수정: icons.tsx(`sparkles`), constants.ts, game-content.ts, [slug]/page.tsx

## Phase별 의존성

```
Phase 1 (결과 공유) ─── 독립
Phase 2 (타이핑) ─── Phase 1 권장 (ShareResult 사용)
Phase 3 (도구 섹션) ─── 독립
Phase 4 (수학 게임) ─── Phase 1 권장 (ShareResult 사용)
Phase 5 (연결 강화) ─── Phase 3 필요 (도구 연결)
Phase 6 (데일리) ─── 독립
Phase 7 (알고리즘) ─── Phase 3 필수 (/tools 라우트)
Phase 8 (퀴즈) ─── Phase 1 권장 (ShareResult 사용)
```

## 기술적 결정 사항

1. **공유 방식**: 클립보드 텍스트 복사로 시작 (zero dependency). 이미지 생성은 추후 필요 시 추가
2. **알고리즘 시각화 위치**: `/tools`에 배치 (점수/등급이 없어 게임보다 도구에 적합)
3. **데일리 챌린지 저장**: localStorage 사용 (크로스 디바이스 불가, 인증 시스템 없으므로 충분)
4. **수학 게임 나눗셈**: b와 answer를 먼저 생성하여 항상 정수 결과 보장
5. **도구 컴포넌트**: 모두 클라이언트 사이드 전용, 백엔드 불필요
