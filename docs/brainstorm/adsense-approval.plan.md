## PDCA-1: 유해 콘텐츠 제거

> **우선순위**: P0 (즉시)
> **의존성**: 없음 (독립 실행)
> **예상 시간**: 2시간

### 목적
애드센스 심사에 치명적인 부정적 신호를 제거한다.

### 범위

| 작업 | 상세 | 완료 기준 |
|------|------|----------|
| Ads 섹션 라우트 삭제 | `app/ads/` 디렉토리 제거 | `/ads` 접근 시 404 |
| 네비게이션에서 Ads 링크 제거 | Header, Footer, 홈페이지 등 | Ads 관련 링크 0개 |
| 광고 플레이스홀더 코드 제거 | 게임 페이지 등의 빈 광고 배너 | 광고 관련 빈 컴포넌트 0개 |
| sitemap에서 /ads 제외 확인 | 크롤러가 접근하지 않도록 | sitemap에 /ads 경로 없음 |

### PDCA 사이클
- **Plan**: 이 문서 (현재)
- **Design**: 별도 설계 불필요 (삭제 작업)
- **Do**: 코드에서 제거 실행
- **Check**: 빌드 성공 + 로컬에서 /ads 404 확인
- **Act**: 문제 없으면 완료

## PDCA-2: 기술 SEO 인프라 구축

> **우선순위**: P0 (즉시)
> **의존성**: PDCA-1 이후 권장 (병렬 가능)
> **예상 시간**: 8시간

### 목적
구글봇이 사이트의 모든 콘텐츠를 정확히 인식하고 인덱싱할 수 있는 기술 기반을 구축한다. 이후 추가되는 모든 콘텐츠가 자동으로 SEO 혜택을 받는다.

### 범위

| 작업 | 상세 | 완료 기준 |
|------|------|----------|
| sitemap.xml 동적 생성 | `app/sitemap.ts` 구현 (블로그 + 게임 + 정적 페이지) | `/sitemap.xml` 접근 시 모든 페이지 URL 포함 |
| robots.txt 최적화 | `app/robots.ts` 구현 (Allow: /, Disallow: /api/) | `/robots.txt` 정상 응답 |
| 메타태그 체계화 | 모든 페이지에 고유 title, description, OG 태그 | 각 페이지 `<head>`에 고유 메타데이터 |
| Schema.org JSON-LD | Article(블로그), SoftwareApplication(게임), Organization(사이트) | Google Rich Results Test 통과 |
| canonical URL | 모든 페이지에 canonical 설정 | `<link rel="canonical">` 존재 |
| 이력서 hreflang | 다국어 이력서 페이지 간 hreflang 태그 | 5개 언어 상호 참조 |

### PDCA 사이클
- **Plan**: 이 문서
- **Design**: `docs/02-design/features/seo-infrastructure.design.md` 작성 필요
- **Do**: Next.js App Router 규약에 맞게 구현
- **Check**: Google Rich Results Test, Lighthouse SEO 점수 90+
- **Act**: 누락된 페이지나 메타데이터 보완

## PDCA-3: 게임 페이지 콘텐츠 보강

> **우선순위**: P0 (즉시)
> **의존성**: PDCA-2 완료 후 (Schema.org, 메타태그 인프라 활용)

6개 게임 페이지의 "씬 콘텐츠" 판정을 해소한다. 각 게임 페이지가 독립적으로 검색 가치 있는 콘텐츠를 제공하도록 보강한다.

| 게임 | 추가할 콘텐츠 | 목표 텍스트량 |
|------|-------------|-------------|
| Reaction Test (반응속도) | 게임 소개, 규칙, 반응속도의 과학, 점수 해석, FAQ | 1500자+ |
| Color Sense (색감) | 게임 소개, 규칙, 색각의 원리, 난이도 설명, FAQ | 1500자+ |
| Color Memory (색상기억) | 게임 소개, 규칙, 기억력과 인지과학, 레벨 가이드, FAQ | 1500자+ |
| Dice Roller (주사위) | 게임 소개, 확률 이론, 활용법, FAQ | 1000자+ |
| Lotto Pick (로또) | 게임 소개, 번호 생성 알고리즘, 확률 설명, FAQ | 1000자+ |
| Animal Face (동물상) | 게임 소개, AI 인식 원리, 사용법, FAQ | 1000자+ |

### 페이지 구조 (모든 게임 공통)

```
[H1: 게임 제목]
[한 줄 소개]
[게임 컴포넌트] ← 클라이언트 렌더링 (기존)
────────────────────────────────
[H2: 게임 소개] ← 서버 렌더링 (새로 추가, 항상 보임)

[H2: FAQ] ← 아코디언

## PDCA-5: 기존 블로그 포스트 보강 + E-E-A-T 신호

> **우선순위**: P1 (필수)
> **의존성**: PDCA-2 완료 후 (메타태그, Article 스키마 활용)
> **예상 시간**: 16시간

### 목적
기존 블로그 7개 글의 길이와 품질을 애드센스 기준까지 끌어올리고, E-E-A-T 신뢰 신호를 추가한다.

### 범위

#### 글 보강 (각 1500자+ 목표)

| 포스트 | 현재 | 보강 방향 |
|--------|------|----------|
| Hello World | ~29줄 | 사이트 소개 확장, 기술 스택 상세 설명, 향후 계획 |
| 시작하기 가이드 | ~45줄 | 단계별 설명 확장, 스크린샷/코드 예제 추가 |
| Next.js 15 + MDX 블로그 | ~129줄 | 이미 양호, 결론/요약 보강 |
| 스픽 100일 사용 후기 | ~64줄 | 비교 분석, 점수표, 추천 대상 확장 |
| iTerm2 한글 깨짐 | ~66줄 | 원인 분석 심화, 다른 터미널 비교 |
| 불테리어 도복 리뷰 | ~61줄 | 착용 사진 설명, 세탁 팁, 비교표 |
| 조커 도복 12개월 리뷰 | ~80줄 | 월별 변화 상세, 내구성 데이터 |

#### E-E-A-T 신호 추가

| 항목 | 구현 방법 |
|------|----------|
| 저자 프로필 | 블로그 포스트 상단/하단에 저자 이름, 약력, 사진 표시 |
| 작성일/수정일 | MDX frontmatter에 date, updatedAt 추가 + UI 표시 |
| 출처 인용 | 외부 링크, 참고 자료 섹션 추가 |
| 카테고리 재분류 | `lifestyle` 카테고리로 도복 리뷰 분리 |

### PDCA 사이클
- **Plan**: 이 문서
- **Design**: `docs/02-design/features/blog-enhancement.design.md` 작성 필요
- **Do**: MDX 파일 수정 + 저자 프로필 컴포넌트 추가
- **Check**: 각 글 글자 수 확인, E-E-A-T 체크리스트 검증
- **Act**: 글자 수 미달 글 추가 보강

|---|-----------------|----------|----------|

| 2 | "색감 테스트로 알아보는 색각의 세계" | Color Sense | 2500자+ |
| 3 | "기억력과 패턴 인식: 사이먼 게임의 인지과학" | Color Memory | 2500자+ |

| 5 | "로또 번호 생성의 비밀: 진짜 랜덤이란?" | Lotto Pick | 2500자+ |

### 블로그 포스트 템플릿 구조

```
H1: [제목]
├─ 도입 (300자) - 일상 연결, 흥미 유발
├─ H2: 과학적/기술적 배경 (600자) - 연구, 원리 설명
├─ H2: 직접 체험하기 (200자) - 게임 페이지 링크

## PDCA-7: 내부 링크 구조 + 카테고리 재편

> **우선순위**: P2 (권장)
> **의존성**: PDCA-5, 6 완료 후 (링크할 콘텐츠가 있어야 함)
> **예상 시간**: 6시간

### 목적
모든 페이지 간 자연스러운 내부 링크 그물망을 구축하여 사이트 전체의 토피컬 오소리티를 강화한다.

### 범위

| 작업 | 상세 | 완료 기준 |
|------|------|----------|
| 게임 ↔ 블로그 상호 링크 | 각 게임 페이지에 관련 블로그 링크, 블로그에 게임 링크 | 페이지당 2-3개 내부 링크 |
| 블로그 관련 글 추천 | 각 블로그 하단에 관련 글 3개 추천 | 관련 글 컴포넌트 구현 |
| 게임 허브 소개 텍스트 | 게임 목록 페이지에 카테고리별 소개 문구 | 300자+ 텍스트 |
| 블로그 카테고리 재편 | web-tech, brain-training, lifestyle 3개 카테고리 | 카테고리 필터 UI 업데이트 |
| 도복 리뷰 분류 | lifestyle 카테고리로 이동 | 카테고리 태그 변경 |
| 게임 페이지 FAQ | 6개 게임에 각 5개 이상 FAQ | 아코디언 UI로 구현 |

### PDCA 사이클
- **Plan**: 이 문서
- **Design**: 내부 링크 맵 작성
- **Do**: 링크 삽입 + 관련 글 컴포넌트 + 카테고리 수정
- **Check**: 데드 링크 검사, 페이지당 내부 링크 수 확인
- **Act**: 고립된 페이지 연결 보완

### PDCA 사이클
- **Plan**: 이 문서
- **Design**: 체크리스트 (위 표)
- **Do**: 항목별 확인 + 재신청
- **Check**: 재신청 결과 대기 (2-4주)
- **Act**: 추가 거절 시 거절 사유 분석 후 재시작

## 다음 단계

각 PDCA 피처의 **Design** 문서를 순서대로 작성한 후 구현에 들어간다.

```
추천 시작 명령어:
  /pdca plan adsense-harmful-removal     → PDCA-1 시작
  /pdca plan seo-infrastructure          → PDCA-2 시작
  /pdca plan game-content                → PDCA-3 시작
```

> **참고**: PDCA-1은 설계가 필요 없는 삭제 작업이므로 바로 Do 단계로 진입 가능. PDCA-2부터는 Design 문서 작성 후 구현 권장.
