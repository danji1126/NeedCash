## 1. 요약

6개 게임 상세 페이지의 "씬 콘텐츠(thin content)" 문제를 해소했다. 각 게임 페이지에 서버 렌더링되는 정적 텍스트 콘텐츠(게임 소개, 플레이 방법, 결과 해석 가이드, 과학적 배경, FAQ, 관련 게임 추천)를 추가하여 게임당 1,485~1,681자의 한글 콘텐츠가 HTML 소스에 포함되도록 했다. 모든 콘텐츠는 서버 컴포넌트로 렌더링되어 구글봇이 직접 읽을 수 있다.

## 5. 기술 노트

### 서버 컴포넌트 전략
- `GameContentSection`과 `RelatedGames`는 `"use client"` 없는 서버 컴포넌트
- 빌드 시 HTML에 모든 텍스트가 포함되어 구글봇이 직접 파싱 가능
- 기존 게임 컴포넌트(클라이언트)와 충돌 없이 공존

### FAQ 아코디언
- `<details>/<summary>` HTML 네이티브 요소 사용
- JavaScript 불필요, 키보드 접근성(Enter/Space 토글) 자동 지원
- 추가 라이브러리 설치 없음

### 데이터 구조
- `lib/game-content.ts`에 모든 콘텐츠 데이터를 TypeScript 상수로 관리
- `getGameContent(slug)`: slug 기반 콘텐츠 조회
- `getRelatedGames(currentSlug, count)`: 현재 게임 제외 관련 게임 반환
